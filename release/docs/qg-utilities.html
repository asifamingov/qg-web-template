Accessibility

To support our users, we're continuing to implement accessibility features to make our sites easier to use.

Link in new window icon

Accessibility guidelines recommend not opening windows in links. However, if opening a window in a new link is required, the QG Web Template automatically styles the links displayed in the #qg-content section with a new window icon.

This link opens in a new window.

Disabling new window icon feature

If there is a user driven reason for it, you can disable this feature by adding the data attribute data-access-new-window=false to the hyperlink you which to disable accessibility on <a href="#" data-access-new-window="false">

If you would like to turn this feature on for any hyperlink outside of the #qg-content element, add the data-access-new-window=true attribute: <a href="#" data-access-new-window="truee">

Javascript

There is also a script that runs over each page to add accessibility to links opening in a new window.

Firstly, the script adds the following text to the end of the link, displayed to screen readers only:

(Opens in new window)

To prevent this, you can add your own notification to the link: <span class="qg-blank-notice">(Opens in new window)</span>

Secondly the script adds a title to your link if one hasn't already been set: <a href="#" title="Opens in new window">

Disabling accessibility features

This is not recommended.

For testing purposes, or if these features are causing issues with applications built on the QG Web Template, these features can be turned off by adding data-qg-accessibility="false" to the <body> tag of your page: <body data-qg-accessibility="false"/>.

Funnelback indexing

By default, Funnelback will index all content within a web page. Finer control is provided through several mechanisms to exclude certain content from being indexed. It is recommended that publishers add the Noindex tag to non-content components within the web page, such as navigation (i.e. headers, breadcrumbs, ToC's, side nav, footer).

For more implementation guidance on how to add Noindex tags, please refer to the PaDRE (Parallel Document Retrieval Engine) documentation.

hCard

Whenever publishers need to display contact details, they can choose to use a special set of HTML class attributes known as hCard. A hCard allows programs to export contacts in a standardised format so that they can be imported into other email programs.

For more information about the hCard, including implementation instructions, please visit the Microformats Wiki.

Click-to-call links

When email addresses are hyperlinked with the ‘mailto:’ command, customers can create a new email in their default program with the TO field already filled out. Similarly, with a click-to-call link, customers on a phone compatible device can call you with a single click.

To create a click-to-call link, add the ‘tel:’ command before the telephone number in the URL, for example “tel:+61730220001”.
